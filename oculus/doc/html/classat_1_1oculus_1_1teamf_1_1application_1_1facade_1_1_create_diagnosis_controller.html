$createDiagnosis

Description: this method first checks the parameters (because they are all required and can not be null or empty). If something is missing , a new RequirementsUnfulfilledException is thrown. Afterwards a new diagnosis object with the incoming parameters is created, and saved into the given examination protocol. Then the new created diagnosis and the examination protocol are saved into the database. If no exception is thrown, the examination protocol and the diagnosis have been saved correct and an interface of the created diagnosis is returned.
